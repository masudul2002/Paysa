# Documentation Roadmap

## Purpose
Define how the documentation matures from planning through production.

## Milestones
| Version | Phase | Documentation Goal |
| --- | --- | --- |
| v0.1 | Planning | Establish source-of-truth structure and unresolved decisions. |
| v0.2 | MVP Definition | Approve product, architecture, offline, data, and testing docs. |
| v0.3 | Implementation Readiness | Validate repository, package, coding, and release rules. |
| v0.4 | Beta Readiness | Add migration, observability, performance, and QA evidence. |
| v1.0 | Production | Freeze baseline requirements and production release process. |
| v1.x | Cloud Sync | Promote sync assumptions into approved implementation decisions. |

- Review P1 documents before every release candidate.
- Review P2 documents monthly or when planning changes.
